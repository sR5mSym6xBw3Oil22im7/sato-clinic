# AGENTS.md

## 目的

本ファイルは、佐藤医院Webサイトの保守・改修を行う担当者向けの作業指針です。  
本リポジトリでは、既存の情報設計と文言整合性を優先し、不要な変更を避けながら静的サイトを維持します。

## 前提

- 本サイトは HTML / CSS / JavaScript のみで構成する
- 文字コードは UTF-8 で統一する
- データベース、CMS、外部バックエンドは使用しない
- 公開先は GitHub Pages を想定する

## 作業方針

- ユーザーの依頼範囲以外は変更しない
- 既存のデザインテイスト、導線、文言ルールを尊重する
- 既存ページのヘッダ、フッタ、主要導線の整合性を崩さない
- Windows 環境での文字化けに注意し、保存時は UTF-8 を維持する

## 実装ルール

- 新規ページ追加時は既存ナビゲーションの構造に合わせる
- 共通スタイルは `css/style.css` に集約する
- 軽微な画面制御は `js/` 配下の既存スクリプト方針に合わせる
- お知らせデータは `js/news-data.js` の配列構造に合わせる
- フォーム送信はモック実装とし、実送信処理を追加しない

## 文言ルール

- 医院名表記は `佐藤医院` に統一する
- 「予約・お問い合わせ」などの主要導線名は既存表記を踏襲する
- ユーザーから明示された修正内容がある場合は、要件資料よりも最新指示を優先する

## 設計・品質ルール

- レイアウト崩れを避けるため、既存CSSの責務を大きく変えない
- スマホ表示の体感速度に配慮し、初期表示に不要な重い画像は遅延読込を検討する
- アクセシビリティを意識し、画像には代替テキストを付与する
- 外部リンクや埋め込みを追加する場合は、既存挙動との整合性を確認する

## ドキュメント更新ルール

- 要件変更や導線変更を行った場合は `doc/` 配下の文書も必要に応じて更新する
- README は利用者向け、AGENTS.md は保守担当者向けとして役割を分ける

## 禁止事項

- 無断でDBや外部API前提の実装へ変更しない
- 既存ページをまとめて削除・リネームしない
- 依頼されていない大幅なUI刷新を行わない
