Requirements

要件定義書

佐藤医院Webサイトの要件を整理した文書です。

1. プロジェクト概要

案件名佐藤医院 Webサイト制作
構成静的Webサイト一式
利用技術HTML / CSS / JavaScript
文字コードUTF-8
公開先GitHub Pages

2. サイト目的

  • 来院前の患者に対して診療内容、診療時間、所在地、院内の雰囲気を伝える
  • 医院の安心感、親しみやすさ、清潔感をWeb上で表現する
  • お知らせや問い合わせ導線を提供し、来院判断を後押しする

3. 対象ユーザー

  • 地域住民
  • 初診を検討している患者
  • 生活習慣病や一般内科の相談先を探している人
  • 診療時間、アクセス、問い合わせ先を確認したい人

4. 必須要件

  • トップページを中心に、主要情報へ遷移できること
  • 診療内容、お知らせ、当院について、スタッフ紹介、FAQ、アクセス、予約・お問い合わせを掲載すること
  • Google マップをアクセスページに埋め込むこと
  • スマートフォン表示に対応すること
  • フッタは `©佐藤医院` のみとすること
受付時間は「午前 9:00〜12:00、午後 13:00〜18:00」です。

5. ページ要件

ページ 役割 主な内容
index.htmlトップページヒーロー、お知らせ、院長挨拶、診療内容、当院について、スタッフ紹介、FAQ、アクセス概要
news-archive.htmlお知らせ一覧全お知らせ一覧
news-detail.htmlお知らせ詳細クエリ文字列による詳細表示
medical-detail.html診療内容詳細追加の診療項目紹介
about-detail.html当院について詳細理念、院内方針、特徴
faq.htmlFAQよくある質問一覧
access.htmlアクセス診療時間、所在地、マップ、交通案内
reservation.html予約・お問い合わせ電話、メール、問い合わせフォームのモック導線

6. 機能要件

  • お知らせ一覧は JavaScript で最新順表示する
  • お知らせ詳細は URL パラメータから対象記事を判定して表示する
  • 問い合わせフォームは入力内容をメール起票用に整形して表示するモックとする
  • 電話、メール、一部未実装導線は `Comming Soon` ポップアップで対応する
  • ヘッダ導線は各ページで共通の情報設計に沿うこと

7. 非機能要件

  • UTF-8 で統一し、文字化けを防ぐこと
  • スマートフォンでも閲覧しやすいレスポンシブレイアウトとすること
  • 初期表示に不要な画像は遅延読込で体感速度に配慮すること
  • アクセシビリティを考慮し、画像に代替テキストを付与すること
