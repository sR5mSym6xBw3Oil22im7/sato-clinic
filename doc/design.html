Design

設計書

画面設計、ファイル設計、主要JavaScript処理、UI方針を記載します。静的サイトとしての運用性を重視した設計です。

1. 画面構成

区分 画面 補足
トップindex.html各主要セクションを1ページに集約
一覧news-archive.html全お知らせ一覧
詳細about-detail.html / medical-detail.html / faq.html / access.htmlトップからの詳細遷移先
予約reservation.htmlフォームモック、電話・メール導線
機能導線news-update.htmlお知らせ更新の将来導線。現状は Comming Soon

2. 共通UI設計

  • ヘッダは全ページで共通メニュー構成とする
  • ロゴ押下時はトップ先頭へ戻る導線とする
  • フッタは全ページ `©佐藤医院` のみ表示する
  • CTA ボタンは主要導線と補助導線でスタイルを使い分ける

3. ファイル設計

ファイル 役割
css/style.css全ページ共通スタイル、レスポンシブ対応、カード・ボタン・ポップアップデザイン
js/news-data.jsお知らせデータ配列を保持
js/news.jsお知らせ一覧表示、詳細表示、日付整形
js/contact.js問い合わせフォームの入力整形とメール起票リンク生成
js/news-update-popup.jsニュース更新ポップアップの表示制御

4. JavaScript設計

お知らせ描画

  • window.SATO_NEWS をデータソースとする
  • 日付降順に並べ替えてトップと一覧へ描画する
  • 詳細ページは ?slug=... で対象記事を判定する

問い合わせモック

  • フォーム送信時にバリデーションを行う
  • 入力内容をもとに件名・本文を組み立てる
  • メールソフト起動用の mailto: リンクを生成する

ポップアップ

  • 電話、メール、お知らせ更新などの未実装導線で `Comming Soon` を表示する
  • 背景クリックまたは閉じる操作で非表示にする

5. レスポンシブ設計

  • CSS Grid と Flexbox を併用する
  • スマホ幅では2カラム系セクションを1カラム化する
  • スマホ初期表示の体感速度向上のため、下層画像は遅延読込を採用する

6. 画像・アセット設計

  • ロゴ、スタッフ写真、院内写真、診療画像を img/ 配下で管理する
  • ファーストビュー画像は優先読込し、それ以外は必要に応じて後読込とする
  • 画像比率は既存CSSの責務で制御し、レイアウト崩れを抑える
